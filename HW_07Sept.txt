1. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
git checkout File1.txt (отменяем изменения файла File1.txt)
git checlout . (отменяем все изнения во всех файлах)


2. Как вернуть файл из индекса в рабочий каталог?
git restore --staged File1.txt (например File1.txt)


3. Как отменить ошибочный commit ?
git reset HEAD-2 (отменит назад два коммита, например)


4. Как исправить коммит отправленный на сервер GitHub?
git commit --amend -m "Add 3 lines"


5-1. Как создать новую ветку и переключится в нее?
Первый вариант:
git branch feature2 - создаём ветку feature2
git checkout feature2 - переключаемся на ветку feature2
Второй вариант:
git checkout -b feature2 - сразу создаем и переключаемся на ветку feature2


5-2.В чем отличение между fast-forward merge  and non fast-forfard merge?
non-fast-forward merge это когда одна ветка отделена от другой ветки. fast-forward merge это одна прямая ветка 


6. Когда может возникнуть конфликт слияния веток, как  его разрешить?
Конфликт может возникнуть, когда в двух ветках была, например, изменена одна и та же строка в файле. Или когда какой-то файл удалён в одной ветке, а отредактирован в другой ветке. Решить конфликты можно отменив слияние веток командой git merge -abort. Можно удалить все конфликты слияния и повторить попытку. 



7. Можно ли удалить ветку? Как? 
Для начала необходимо переключиться на другую ветку, например, feature1, используя команду:
git checkout feature1
Далее удаляем ветку feature2:
git branch -d feature2
